Object Oriented Programming After a Few Years Writing Functional Code

I signed up for a full stack bootcamp to combine a few interests of mine: visual design, data analysis and modern coding practices. Web applications connect the user interface with the server and database using software and hardware that is constantly evolving. Object Oriented Programming (OOP) is a style of coding essential to web development and has been passed on while languages and tools are constantly undergoing change. Increasing my use of this style of programming has been an essential development as someone who writes code. My first exposure to software development was evaluating mathematical functions. My B.S. Thesis was a simple time series algorithm and I have been hooked on writing code ever since. At the time I expected my research would be a stand alone project. I had no idea I would be adapting the methodology through a M.S. Thesis and a publication. Self admittedly, I was guilty of separating related data and functions with reuse of similar interrelated functions with some of interdependency. Learning Full Stack Development has enhanced my use of OOP to improve the clarity and reusability of my code.

OOP takes out unnecessary interdependency with four major principles, Encapsulation, Abstraction, Inheritance and Polymorphism.


Encapsulation

OOP combines groups of related variables and functions into objects with properties and methods. This reduces parameters and makes it easier to use and maintain functions. The coding sample below shows an example of functional programming in JavaScript. The variables are declared and passed into the function as parameters. 

// Transportation Costs Including Gas and Regular Maintenance                         
let price = 20000;
let maintenancePerMonth = 60;
let gasPerMonth = 150;
let monthsOfOwnership = 120;

funciton getTransportationCost(price, maintanencePerMonth, gasPerMonth, monthsOfOwnership) {
  return price + ((maintenancePerMonth + gasPerMonth) * monthsOfOwnership);
}

The variables and the functions are decoupled. If someone was to make revisions to the code the relationships would be less clear than if the parameters were modeled as properties of an object. This further improves organization when the amount of properties and methods increase. The same operations are listed below using OOP. Think about how much clearer this code would be to read if there was a group of people comparing cost of ownership (assuming they drive the same amount). 

// Transportation Costs Including Gas and Regular Maintenance                                                 
let vehicle = {
  price: 20000,
  maintenancePerMonth: 60,
  gasPerMonth: 150,
  monthsOfOwnership: 120,

  getTransportationCost: function() {
    return this.price + ((this.maintenancePerMonth + this.gasPerMonth ) * this.monthsOfOwnership);
  }
}

This JavaScript object could be made with a Java class as well. Classes are blueprints for objects. An example of the Vehicle class would be as follows: 

public class Vehicle {
  public final double transportationCost;
  public double price;
  public double maintenancePerMonth;
  public double gasPerMonth;
  public double monthsOfOwnership;
              
  public double calculateTransportationCost(double price, double maintenancePerMonth, double gasPerMonth, double monthsOfOwnership) {
    return this.price + ((this.maintenancePerMonth + this.gasPerMonth ) * this.monthsOfOwnership);
  }
              
  Vehicle(double price, double maintenancePerMonth, double gasPerMonth, double monthsOfOwnership) {
    this.price = price;
    this.maintenancePerMonth = maintenancePerMonth;
    this.gasPerMonth = gasPerMonth;
    this.monthsOfOwnership = monthsOfOwnership;
              
    this.transportationCost = calculateTransportationCost(price, maintenancePerMonth, gasPerMonth, monthsOfOwnership);
  }
              
  double getTransportationCost(){
    return this.transportationCost;
  }
              
}

Abstraction

Abstraction is separating properties and methods from the outside code. It is easier to understand an object with a few properties than an object with a lot of properties and methods. Abstraction helps create a simpler user interface and reduces the impact of change. The main method listed below could perform the transportation cost comparison. Someone reading the main class would immediately know the comparison being made and could look in the Vehicle class to see what each of the parameters are. 

public static void main(String[] args){
  Vehicle vehicle = new Vehicle(20000, 60, 150, 120);
  Vehicle hybridVehicle = new Vehicle(25000, 50, 80, 120);

  System.out.printf("My vehicle cost %f, but if it was a hybrid it would cost %f",
          vehicle.getTransportationCost(), hybridVehicle.getTransportationCost());
}

This process could be repeated in JavaScript by creating a hybridVehicle object with the same fields and methods but different values such as lower gas and maintanence expenses and a higher upfront cost. The methods vehicle.getTransportationCost() and hybridVehicle.getTransportationCost(). In both coding languages the code comparing the two vehicles reads similar to plain English.

Inheritance

Inheritance is a mechanism in which one class acquires the property of another class. This eliminates redundant code between different classes. A popular use of inheritance is child classes, which can be made to inherit the data and code features of their parent class. This is key to web development with JavaScript libraries such as React JS which allows for reusing and modifying components that make up the UI. Functionality common to the components can be reused eliminating redundant code.

Inheritance can be applied to the Java example by creating a CompanyVehicle subclass of the vehicle superclass. This vehicle has maintanence reimbersed by the company so it has been removed from the Transportation costs. The subclass was created with only the declaration and the calculateTransportationCost override because the features that are the same were inherited using the extends keyword. 

public class CompanyVehicle extends Vehicle {

  CompanyVehicle(double price, double maintenancePerMonth, double gasPerMonth, double monthsOfOwnership) {
    super(price, maintenancePerMonth, gasPerMonth, monthsOfOwnership);
  }

  public double calculateTransportationCost(double price, double maintenancePerMonth, double gasPerMonth, double monthsOfOwnership) {
    return this.price + (this.gasPerMonth * this.monthsOfOwnership);
  }
    
}

Polymorphism

Polymorphism is a technique for getting rid of the long conditional statements associated with reusing functions. A methodology can be used in each object with differences handled in the object rather than a set of conditions within a function. Going back to the vehicle Java example, an array could be made to compare the costs of ownership if the person uses their personal vehicle for work of the vehicle using a polymorphic array. 

public static void main(String[] args){
  Vehicle costComparison[] = new Vehicle[2];
  costComparison[0] = new Vehicle(20000, 60, 150, 120);
  costComparison[1] = new CompanyVehicle(20000, 60, 150, 120);

  for (int i=0; i<=costComparison.length-1; i++){
    System.out.println(costComparison[i].getTransportationCost());
  }

}

OOP is Another Way to Keep Improving

I do not necessarily believe that there is one correct style of writing code or even that my approach was incorrect when writing the procedure for my research. Now that I have written enough OOP code I can tell my transition to web development will be heavy with OOP. Taking time to attend a full stack bootcamp has been rewarding for myself (and anyone looking to read or reuse my codes). OOP is very large subject and this brief blog post has only covered the four most fundamental principles at the surface level. I would encourage anyone interested to go much deeper into the topic in their language of choice.
